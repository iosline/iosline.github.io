Android Studio的一些坑

发表于 2016-11-14   |     |   阅读次数

最近由于项目中的方法数过多，经常出现65535的异常，eclipse打包不成功，迫不得已，将原来的项目转到android studio上来继续开发，先后遇到了这样或者那样的问题。来记录一下：

eclipsetostudio

从Eclipse中导出Gradle build files

在Eclipse菜单中 File –> Export–>Generate Gradle build files

修改导出文件参数(或者导入studio之后在studio中修改)

导出后，由于adt很久没更新，需要手动改一些参数，才能保证正常使用。
分情况说一下：

  • 没有库工程，只有主工程：module

    • 修改主工程的build.gradle 文件，修改需要的gradle版本

    在 /gradle/wrapper/gradle-wrapper.properties里面,更新gradle版本

  • 含有库工程： project

    • 和上面一样，只不过修改的是外面整个project的build.gradle和/gradle/wrapper/gradle-wrapper.properties。

    导入AndroidStudio

    这时打开 studio open 或 import 即可。

    6.Manifest merger failed with multiple errors, see ？

    Error:Execution failed for task ‘:app:processDebugManifest’.

    Manifest merger failed with multiple errors, see logs

    原因是导入的库在build.gradle中的minSdkVersion与你的应用的minSdkVersion不匹配导致的(app要求应用最小系统版本和库要求系统最小版本不一致),改成一样的就行了.

    7.warning: Ignoring InnerClasses attribute for an anonymous inner class

    that doesn’t come with an
    associated EnclosingMethod attribute. This class was probably produced by a
    compiler that did not target the modern .class file format. The recommended
    solution is to recompile the class from source, using an up-to-date compiler
    and without specifying any “-target” type options. The consequence of ignoring
    this warning is that reflective operations on this class will incorrectly
    indicate that it is not an inner class.

    解决办法：

    好像是 debug 签名过期了 ；
    去 Users/用户名/.adnroid 下，删除：debug.keystore和ddms.cfg 即可。

    8.SVN 找不到了

    今天早上开机，打开android studio 的时候，突然发现，自己的studio中的subversion找不到了。前几天还用的好好的，今天突然找不到了，我很是纳闷了。再网上找了很多的资料，大部分都说你的svn 安装的不对，后来请教了大神，知道了错误在哪里？原来是plugin里边的subversion没有勾选啊。

    9.打包时遇见

    Error:A problem was found with the configuration of task ‘:app:packageRelease’.

    File ‘/Users/jiayongkai/Downloads/jdandroid/app/build/intermediates/res/resources-release-stripped.ap‘ specified for property ‘resourceFile’ does not exist.

    解决办法：

  • 1、instant run 不支持 shrinkResources File –> Settings–>instant 取消第一个选项

  • 2. 并将module中的build.gradle中shrinkResources置为false，然后重新编译即可
